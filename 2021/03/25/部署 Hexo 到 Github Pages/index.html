部署 Hexo 到 Github Pages

3月 25 2021 5 分钟 读完 (约 686 字)

创建 GitHub Pages Repository, 下载 Hexo 到本地并配置, 部署 Hexo 到 GitHub Pages 上

GitHub Pages

创建 GitHub Pages Repository

打开自己Github账户的主页 -> Repositories -> New

Repository name 设置为 username/username.github.io // !注意 用户名严格区分大小写

Repository 必须为 Public

Hexo

0. 环境要求

  1. 下载安装 Node.js (建议选择LTS版本) https://nodejs.org/en/

  2. 下载安装 cnpm

1. 安装hexo到本地

2. 写Markdown (可嵌入HTML)

打开 source/_posts, 新建 xxx.md 或者 使用 hexo new xxx

在其中写Markdown

3. 设置主题 (可选)

  1. 安装主题所需的依赖
  2. 在theme文件夹下新建一个文件夹, 然后把下载的主题放进去
  3. 修改根目录下的 _config.yml theme字段为主题所放置的文件夹的名字
  4. 修改主题目录下的 _config.yml.example 为 _config.yml, 然后修改里面的内容即可实现配置主题

4. 本地测试

开启本地服务器, 查看效果

5. 设置域名 (可选)

首先你要去域名服务商哪里购买一个域名
比如: 万网(推荐) 西部数码 GoDaddy …

域名服务商端设置

…略

Hexo端设置

在source目录下创建一个 CNAME 文件

写入 www.二级域名.顶级域名

6. 部署准备

安装 hexo-deployer-git

配置 _config.yml

7. 部署

8. 将GitHub Pages Repository的默认Branch更换成_config.yml 中 deploy 中 branch 的值

7. 如果刷新没生效

Shift+F5 // 清除缓存并刷新
